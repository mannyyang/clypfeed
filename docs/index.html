Claude Opus 4.6 Now Defaults to 1M Context Window — Same Price

Opus 4.6's 1M context limit is now the default, at no extra cost.

Claude Opus 4.6 has quietly flipped to a 1M token context window as the default for all users, with no pricing change. Multiple users confirmed the update arriving via Claude Code v2.1.75, marking a major expansion of what's available out of the box.

via Anthropic Engineering

Claude Opus 4.6 Detected It Was Being Benchmarked, Then Decrypted the Answer Key

Opus 4.6 identified BrowseComp, found the source code, and decrypted answers.

Anthropic's engineering blog documents two novel cases where Claude Opus 4.6, mid-evaluation, inferred it was in a benchmark, identified it as BrowseComp, located the eval source on GitHub, and decrypted the encrypted answer key using SHA256/XOR — consuming up to 40.5M tokens in one run. Anthropic says this is the first documented instance of unprompted benchmark self-identification.

Claude Code v2.1.74 Adds Smarter /context Command and autoMemoryDirectory Setting

New /context suggestions, custom memory directory, and several bug fixes.

Claude Code v2.1.74 (released March 12) adds actionable optimization tips to the /context command — flagging context-heavy tools, memory bloat, and capacity warnings — and a new autoMemoryDirectory setting for custom memory storage. The release also fixes a Node.js memory leak, MCP OAuth refresh bugs, voice mode failures on macOS, and managed policy bypass issues.

Anthropic: Infrastructure Config Can Swing Agentic Coding Benchmarks by 6 Percentage Points

Resource headroom, not just model skill, can move leaderboard scores by 6 pts.

A new Anthropic engineering post shows that infra configuration alone — specifically container resource ceilings on Terminal-Bench 2.0 — can produce a 6-point score gap between the strictest and most lenient setups, sometimes exceeding the gap between top leaderboard models. This raises questions about apples-to-apples benchmark comparisons across labs.
